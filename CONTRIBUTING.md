# Contributing to Awesome Talon

Thank you for your interest in contributing to Awesome Talon!

## Guidelines

- Search previous entries before making a new one to avoid duplicates.
- Make an individual pull request for each suggestion.
- Use the following format: `[Name](link) - Description.`
- New categories or improvements to the existing categorization are welcome.
- Keep descriptions short and simple, but descriptive.
- Start the description with a capital letter and end with a period.
- Check your spelling and grammar.
- Make sure your text editor is set to remove trailing whitespace.
- The pull request should have a useful title.

## What qualifies?

Anything that is relevant to the Talon Voice ecosystem:

- Voice command sets and plugins
- Editor/IDE integrations
- Browser extensions
- Accessibility tools
- Learning resources and documentation
- Blog posts, talks, and tutorials
- Hardware integrations
- Developer tools for building Talon scripts

## Updating your Pull Request

If the maintainers notice anything that we'd like changed, we'll ask you to edit your PR before merging it. If you're not sure how to do that, [here is a guide](https://github.com/RichardLitt/knowledge/blob/master/github/amending-a-commit-guide.md) on the different ways you can update your PR.
